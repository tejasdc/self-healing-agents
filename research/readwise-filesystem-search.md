## Key Themes for Self-Healing Agent Design

Based on this literature survey, five meta-themes emerge:

1. **Feedback loops, not blueprints:** Living systems heal through feedback loops, not by following a master repair plan. Self-healing agents should be loop-driven (detect -> diagnose -> act -> verify), not rule-driven.

2. **Antifragility through decentralization:** Systems that gain from disorder (Bitcoin, immune systems, mycelial networks) all share decentralization. No single point of failure. Weak components are sacrificed so the system strengthens. Self-healing agents should allow component failure and rebuilding.

3. **Pace layering:** Different healing responses at different timescales. Fast: restart a failed tool call. Medium: adjust strategy for a task. Slow: update long-term behavioral patterns. The slowest layers provide stability; the fastest provide responsiveness.

4. **Stigmergic coordination:** Agents leaving "healing traces" in shared environments (logs, patches, learned context) that other agents or future runs can consume. No centralized healing coordinator needed.

5. **Entropy is the default:** Without active maintenance, all systems degrade. Self-healing is not a feature -- it's the ongoing cost of existence. As the Red Queen says: "It takes all the running you can do, to keep in the same place."
